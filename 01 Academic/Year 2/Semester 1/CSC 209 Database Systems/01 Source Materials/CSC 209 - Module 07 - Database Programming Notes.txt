1. Introduction to Relational Databases
 
Relational databases are the most common way business data is stored and 
organized. A relational database consists of 
tables
 
that store data in a 
structured format with rows and columns.
 

 
Table
: A table in a relational database consists of records (rows) and fields 
(columns).
 

 
Primary Key
: A unique identifier for a record in a table.
 

 
Foreign Key
: A field in one table that links t
o the primary key in another 
table, creating a relationship between tables.
 

 
Relational Model
: It enforces rules about how data can relate across 
tables (e.g., foreign key relationships).

2. Joining and Retrieving Data
 
When data is spread across m
ultiple tables, 
joins
 
are used to combine this data 
into a meaningful result.
 
Types of Joins:
 

 
INNER JOIN
: Retrieves records with matching values in both tables.
 

 
LEFT JOIN
 
(or LEFT OUTER JOIN): Retrieves all records from the left table, 
and matching records
 
from the right table. Returns NULL where there is no 
match.
 

 
RIGHT JOIN
 
(or RIGHT OUTER JOIN): Retrieves all records from the right 
table, and matching records from the left table.
 

 
FULL OUTER JOIN
: Retrieves all records where there is a match in one of the
 
tables, filling NULLs where there are no matches.
 
Example
 
SELECT customers.name, orders.order_id
 
FROM customers
 
INNER JOIN orders ON customers.customer_id = orders.customer_id;
 
 
This query retrieves the customer name and order ID for customers with 
matching records in the 
orders
 
table.
 
Filtering Data:
 
You can apply filters to your SQL queries using the 
WHERE
 
clause:
 
SELECT * FROM orders WHERE order_date > '2023
-
01
-
01';
 
 
Sorting 
and Limiting Results:
 

 
ORDER BY
: Sorts the results based on a column.
 

 
LIMIT
: Restricts the number of rows returned.
 
SELECT * FROM orders ORDER BY order_date DESC LIMIT 10;
