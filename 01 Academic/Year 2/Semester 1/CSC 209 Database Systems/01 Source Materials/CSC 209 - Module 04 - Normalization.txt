Norma
lization 
in a Relational Model
 
   
-
 
Normalization
:
 
The process of organizing data in a database to minimize 
redundancy and improve data integrity.
 
   
-
 
Normal Forms
:
 
Rules that a relational database must follow. Key normal forms 
include:
 
     
-
 
First Normal Form (1
NF)
:
 
Ensures that the values in a table are atomic 
(indivisible), i.e., each column contains unique and individual values.
 
     
-
 
Second Normal Form (2NF)
:
 
Builds on 1NF by ensuring that all non
-
key 
attributes are fully functionally dependent on the primar
y key.
 
     
-
 
Third Normal Form (3NF)
:
 
Ensures that no transitive dependencies exist, i.e., 
non
-
key attributes do not depend on other non
-
key attributes.
 
 
 
Importance of Normalization
 
-
 
Prevents data anomalies
 
like:
 
  
-
 
Insertion anomalies
: Problems with 
adding new data.

--- PAGE 2 ---
-
 
Deletion anomalies
: Problems with deleting data that results in unintended 
data loss.
 
  
-
 
Update anomalies
: Changes in data cause inconsistencies.
 
-
 
Reduces storage space
 
-
 
Improves query performance
