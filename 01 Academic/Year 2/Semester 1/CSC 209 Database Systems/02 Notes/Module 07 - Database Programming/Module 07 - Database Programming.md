## Key Ideas
- Database programming connects relational databases to application code and practical data workflows.
- The source emphasizes SQL querying, joining data from multiple tables, and moving data using formats such as CSV and JSON.
- A programmer must understand both relational structure and the commands used to retrieve, insert, update, and delete data.
- Practical work often includes exporting, importing, and transforming data between systems.

## Core Concepts
- Relational databases store data in tables with rows and columns.
- Primary keys uniquely identify records.
- Foreign keys connect tables and enforce relationships.
- SQL is the standard language for interaction with relational databases.

## Join And Retrieval Skills
- Use joins to combine related data across tables.
- Use `WHERE` to filter.
- Use `ORDER BY` to sort.
- Use `LIMIT` to control output size.
- Understand when `NULL` values appear in outer joins.

## Data Interchange
- `CSV` is simple, portable, and common for import/export.
- `JSON` stores data as key-value structures and is useful in programming contexts.
- Database programming often involves moving data between the database and external files or applications.

## Practical Tools Mentioned
- MySQL
- PostgreSQL
- SQLite
- SQL Server
- Python examples for handling CSV and JSON
