WHY DATA WRANGLING

In real life applications, data comes from different sources in different formats. There is always a need to rearrange, combine, reshape, filter the data before carrying out any analysis on it. In this section we are going to be looking at the techniques for carrying out this particular tasks. What is data wrangling

The process of changing, or transforming data from one particular shape to another particular shape is usually reffered to us data wrangling. Some people refer to this process as data mugging. Data wrangling involves restructuring, cleaning and converting it into a format that makes it easy to analayse the data.

Combining dataframes: Pyhton offers several methods for combining dataframes. You can combine dataframes using the merge() function, the join function and the concatenate function.

Join

Sometimes you might have data in different files. You can load the data into two different dataframes and use the join method to combine the two dataframes. The join function is used when you have a common column or a common index. It is more easy to use the join function using an index rather than a column.

The syntax for the join method is as follows;

    DataFrame.join(other, on=None, how='left', lsuffix='', rsuffix='', sort=False)

Parameters

other: This is the name of the the other dataframe to be combined

how: This indicates the type of the join to be performed. There are different types of joins that will result in different dataframes.

  1. left: Use the keys found in the first dataframe. Get all the things from the left dataframe and only the matching values from the right(other) dataframe

  2. right: Use the keys found in the other dataframe. Gets all the values from the right dataframes and only those that match in the right daaframes.

  3. outer: Uses the keys from both dataframes, returns both the elements in both dataframes

  4. inner: Uses the intersection of keys from both dataframes, it returns only the data elements that are have the same keys in both dataframes

sort: Indicates if the resultant dataframes will be sorted. A value of "false" will indicate that the dataframe will not be sorted and a value of "True" will sort the resultant dataframe.

rsuffix: This is the suffix to be used for overlapping columns

concat() function

The other method provided in pandas for combining two dataframes are the concat() function. As opposed to join and merge functions, concat can combine more that two dataframes. The focus of the concat method is to combine data along the axis i.e the rows and the columns.

The syntax for concat function is as follows

    pd.concat(objs, axis=0, join='outer', ignore_index=False, keys=None, levels=None, names=None, verify_integrity=False, sort=False, copy=True)

objs: are the dataframes or the series to be combined.

axis: Determines the axis a long which data is combined.0 Indicates a long the rows and 1 indicates a long the column. Along the rows stacks the rows of the dataframes together. Along the axis, stacks the collumns of the provided dataframes together.

verify_integrity: Check whether the new concatenated axis contains duplicates

Pivoting

Pivoting is a data transformation process commonly used in data analysis to reshape data. In pandas, pivoting involves converting a DataFrame from a long or tall format (where rows represent individual observations) to a wide format (where each unique value in a column becomes a new column). This is useful for creating summary tables and for visualizing data more effectively.

Pivot: This creates a new DataFrame by rearranging the data such that unique values from one column become the columns of the new DataFrame, and values from another column become the rows.

Pivot Table: This is Similar to a pivot, but it allows for the aggregation of data. For example, it can sum, average, or count values within each group of the new table.

Example

To demonstrate this concepts lets create a simple data frame below

Stacking

Stacking is another data transformation process in pandas, used to convert a DataFrame from a wide format to a long format. It's the opposite of unstacking or pivoting, where you might go from a long format to a wide format. Stacking is particularly useful when you need to perform operations on data that is easier to handle in a long format or when preparing data for certain types of analyses or visualizations.

There are two main concepts in stacking: Stacking and Unstacking

Stacking: Stacking reshapes the dataframe by converting the data into a stacked form. This involves creating a pivot of the innermost column index into the innermost row index.

Example

import pandas as pd

data = {
    'date': ['2024-06-01', '2024-06-01', '2024-06-02', '2024-06-02'],
    'North_sales': [100, 200, 300, 400],
    'South_sales': [150, 250, 350, 450]
}
df = pd.DataFrame(data)

stacked_df = df.set_index('date').stack()
print(stacked_df)

Unstacking

Unstacking reverses the stacking process

Example

import pandas as pd

data = {
    'date': ['2024-06-01', '2024-06-01', '2024-06-02', '2024-06-02'],
    'North_sales': [100, 200, 300, 400],
    'South_sales': [150, 250, 350, 450],
    'North_quantity': [10, 20, 30, 40],
    'South_quantity': [15, 25, 35, 45]
}
df = pd.DataFrame(data)

# Reset the index to ensure uniqueness
df.reset_index(inplace=True)

# Set a new unique index
df.set_index(['index', 'date'], inplace=True)

# Perform stacking
stacked_df = df.stack()
print(stacked_df)

unstacked_df = stacked_df.unstack()
print("\nUnstacked DataFrame:\n", unstacked_df)

Key things to note

The query() method only works if the DataFrame’s column names are valid Python identifiers (no spaces or special characters). If not, use backticks to enclose column names.

When using variables inside the query string, prefix them with @.

String operations like .str.contains() can be used within query() by referring to the column as a string attribute.
