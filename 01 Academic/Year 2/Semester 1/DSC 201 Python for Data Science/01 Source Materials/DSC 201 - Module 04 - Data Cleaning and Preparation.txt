After you have indetified that your dataset has null or missing values, then you can now look at the strategies for handling the missing values. Methods for handling missing values Droping the missing values One of the ways of handling the missing values is removing the entries with missing values. This is usually reffered to as dropping the missing values. Pandas provides a function called dropna() for removing the missing values. Parameters for the dropna() function axis : The axis value determines if it is a row or a column of the missing value will be removed. A value of 0 for the axis means remove the rows that have missing values. A value if 1 means remove the column that has missing values. how : The how determines when the row or column is removed from the axis. The option are "any", if we have any Null value remove the column or row and "all", if all the values in the column or row are null remove the column or row. You can strat by removing the fully empty columns and rows and then look for other strategies for filling in the values where it is not an entire row or column that is empty. inplace : This determines if the original dataset will be modified or a copy will be created with the modified values. A value of True means that the original dataset is modified, a value of False means that the original dataset will not be modified. There are other parameters you can set for the dropna() function, be sure to check the documentation. Lets demonstrate here the use of dropna() function in our dataset loaded above new_data = data . dropna ( axis = 0 , inplace = False ) new_data . isnull () . sum () TK104 0 TK105 0 TK107 0 dtype: int64 We can see our new dataset has 0 null values. We can check the the information about the dataset using the info() method as follows new_data . info () Index: 18 entries, 0 to 22 Data columns (total 3 columns): # Column Non-Null Count Dtype --- ------ -------------- ----- 0 TK104 18 non-null float64 1 TK105 18 non-null float64 2 TK107 18 non-null int64 dtypes: float64(2), int64(1) memory usage: 576.0 bytes We can see that after dropping the null values our new dataset has reduced to 18 entries. Imputing the missing values The other way of handling the missing values is replacing it with a value. Depending on the application area and the snesitivity of the data, missing values can be replaced with a common value. Pandas provides a function called fillna() which can be used to replace the values of the missing data with a common value. Parameters for fillna() function The syntax for the fillna() function is as follows DataFrame.fillna(value=None, method=None, axis=None, inplace=False, limit=None, downcast=None) value : This specifies the value that the null values will be filled with. method : This specifies the method to use to fill the holes if there was reindexing. The options for method are ‘backfill’, ‘bfill’, ‘ffill’, None. The default is usually none. The 'backfill','bfill' , method fills the gap using the next valid observation. ffill : Propagate last valid observation forward to next valid. axis : This specifies the axis either row or column that the value will be filled with inplace : Specifies whether the original dataframe will be changed will or a copy will be created. setting inplace to True the dataframe will be changed and if false a copy will be created. limit : This is the maximum number of consecutive NaN values to forward or backward fill, provided a method is supplied. Put otherwise, a gap will only be partially filled if there are more consecutive NaNs than this number. This is the maximum number of entries along the entire axis where NaNs will be filled, if method is left unspecified. If not None, it must be greater than 0. downcast : dictionary of item data type of what to downcast if possible. For example, downcast={'float': 'int'} will attempt to downcast floats to integers Example Let us use the fillna() function to fill the values that import pandas as pd data = pd . read_csv ( "C:/Users/User/Dropbox/OUK Documentts/Python for data science module development/Datasets/brittleness-index.csv" ) data . head () TK104 TK105 TK107 0 254.0 263.0 338 1 440.0 NaN 470 2 501.0 NaN 558 3 368.0 451.0 426 4 697.0 709.0 733 new_data = data . fillna ( value = 10 , limit = 2 ) new_data . head () TK104 TK105 TK107 0 254.0 263.0 338 1 440.0 10.0 470 2 501.0 10.0 558 3 368.0 451.0 426 4 697.0 709.0 733 As you can see the values at index 1 and 2 have been filled with the values 10. Depending on what kind of analysis you want to carry, having duplicates might not be desirable so you may need to find out if your data has any duplicates and and how to handle them.
