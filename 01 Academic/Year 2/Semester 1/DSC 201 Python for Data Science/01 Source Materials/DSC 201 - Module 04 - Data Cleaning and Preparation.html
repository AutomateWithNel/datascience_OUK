Whats is data cleaning

Data cleaning is a process in the data science that helps in making sure that you have accurate, consistent and a complete dataset. Data cleaning and preparation are usually the inital tasks that any data scientist will have to carry out at the beginning of any data science projects.

The most amount of time in datascience projects is used in doing data cleaning and preparation. The major tasks done during this stage include; loading the data, cleaning the data,tranforming and rearanging the data. In the previous module we looked at the various sources of data and how we can extract the data from those sources. Once this data is extracted, you might be required to merge different data sources and you will need to clean the data to ensure that you have complete and consistent data.

Why is it necessary to clean and prepare data?

  1. Data is not always in the right format for analysis
  2. Data is not always clean. It can contain missing data, duplicates and so on
  3. Data is also cleaned to ensure that it conforms to a certain range
  4. Having clean data enhances the perfomace of machine learning algrorithms.

This are just a few reasons for doing data cleaning. Can you think of any other?

Now that we know how important data cleaning and preparation is, lets delve into some of the strategies we can use to clean and prepare our data for data analysis.

Methods for handling missing values

Droping the missing values

One of the ways of handling the missing values is removing the entries with missing values. This is usually reffered to as dropping the missing values. Pandas provides a function called dropna() for removing the missing values.

Parameters for the dropna() function

axis: The axis value determines if it is a row or a column of the missing value will be removed. A value of 0 for the axis means remove the rows that have missing values. A value if 1 means remove the column that has missing values.

how: The how determines when the row or column is removed from the axis. The option are "any", if we have any Null value remove the column or row and "all", if all the values in the column or row are null remove the column or row. You can strat by removing the fully empty columns and rows and then look for other strategies for filling in the values where it is not an entire row or column that is empty.

inplace: This determines if the original dataset will be modified or a copy will be created with the modified values. A value of True means that the original dataset is modified, a value of False means that the original dataset will not be modified.

There are other parameters you can set for the dropna() function, be sure to check the documentation.

Lets demonstrate here the use of dropna() function in our dataset loaded above

Imputing the missing values

The other way of handling the missing values is replacing it with a value. Depending on the application area and the snesitivity of the data, missing values can be replaced with a common value. Pandas provides a function called fillna() which can be used to replace the values of the missing data with a common value.

Parameters for fillna() function

The syntax for the fillna() function is as follows

    DataFrame.fillna(value=None, method=None, axis=None, inplace=False, limit=None, downcast=None)

value: This specifies the value that the null values will be filled with.

method: This specifies the method to use to fill the holes if there was reindexing. The options for method are ‘backfill’, ‘bfill’, ‘ffill’, None. The default is usually none.

  1. The 'backfill','bfill', method fills the gap using the next valid observation.

  2. ffill: Propagate last valid observation forward to next valid.

axis: This specifies the axis either row or column that the value will be filled with

inplace: Specifies whether the original dataframe will be changed will or a copy will be created. setting inplace to True the dataframe will be changed and if false a copy will be created.

limit: This is the maximum number of consecutive NaN values to forward or backward fill, provided a method is supplied. Put otherwise, a gap will only be partially filled if there are more consecutive NaNs than this number. This is the maximum number of entries along the entire axis where NaNs will be filled, if method is left unspecified. If not None, it must be greater than 0.

downcast: dictionary of item data type of what to downcast if possible. For example, downcast={'float': 'int'} will attempt to downcast floats to integers

Transforming data

In some instances in your datascience project, you might need to transform the dataset into a certain format, rename columns to avoid confusion, aggregate the data and so on. In this section, we are going to look at some of the methods you can use to transform your data.

Removing duplicates

In some instances maybe due to combining data from different sources, you might have duplicate values in the dataset. Depending on what kind of analysis you want to carry, having duplicates might not be desirable so you may need to find out if your data has any duplicates and and how to handle them.

In order to remove duplicates we you can use the pandas function drop_duplicates(). Ths syntax for the drop_duplicates() function is as follows.

    DataFrame.drop_duplicates(subset=None, keep='first', inplace=False, ignore_index=False)

parameters

subset: This specifies the column to be considered when dropping the deplicates, by default all the columns are considered.

keep: Specifies which duplicate to keep. If the value is set to "first" the first occurance of the data point is kept. If set to "last" the last occurance of the duplicate data point is kept. If set to "False" all the duplicate data points are dropped.

inplace: Specifies whether the dataframe is modified or a copy is created.

ignore_index: Specifies whether the original index in the dataset is kept

Example

data_duplicates = pd.DataFrame({
    'brand': ['Potatoes', 'Potatoes', 'Rice', 'Rice', 'Rice'],
    'style': ['basket', 'basket', 'basket', 'pack', 'pack'],
    'rating': [4, 4, 3.5, 15, 5]
})

data_duplicates.head()

To remove duplicates based on all the column, we do not specify the subset. This will only remove a data point that has duplicate values in all the columns

data_duplicates.drop_duplicates()

To remove duplicates based on a specific column you will need to specify the subset value. This will remove all the duplicate values with the same value in that particular column

data_duplicates.drop_duplicates(subset=['brand'])

To keep an instance of the last instance of the duplicate instance we specify the 'keep' parameter

data_duplicates.drop_duplicates(subset=['brand'], keep='last')
