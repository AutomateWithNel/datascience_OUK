Reading the excel file in pandas

To read excel file in pandas you will use the pd.read_excel(). The read_excel() function expects the following parameters.

Excel file path: Provides the path to the excel file.

Sheet_name: This specefies the sheet name that your data is in this is applicable in cases where the excel file cointains more than one sheet.

Header: The header indicates the row to be used for the names of the column. The default one is the row at index 0.

usecols: Indicates the columns that you will need to use in your dataframe. The default is none which will load all the columns in the dataframe. If you need only specific columns from your excel sheet, then you need to specify them as the list.

The list above is not exhostive of all the parameters that you can set when reading an excel file. For more information you can visit the documentation for pandas on the excel function

https://pandas.pydata.org/docs/reference/api/pandas.read_excel.html

Example reading the excel file

In this example you will see how you can set some of the paremeters while reading an excel file. We have provided the path to the file and the sheet_name

import pandas as pd
data=excel_file=pd.read_excel("Admission_Predict.xlsx", sheet_name="Admission_Predict", header=0)

to_excel(): Pandas also provides a function for writing to excel. This function helps in saving your data as an excel file. The function is called to_excel()

Parameters

The following are the paremeters you need to set in order to write to an excel file

excel_writer: This indicates where to write too. It can be a path or an existing excel writter.

sheet_name: The will indicate the name of the sheet that will cointain the data to be written. The dafault is usually sheet1

float_format: This will indicate how the floats will be formated. For example providing "%.3f" will save the floats in three decimal places

header: It usually expects a boolean or a list of strings. If set to True the heading in the dataframe will be used as headers. If a list of string is provided the headers will be mapped to those strings.

index: This specifies if the index present in the dataframe will be used as indexes in the saved data. If set to fales, the indexes will not be included. The default value is True.

index_label: This is an optional paremeters meaning you do not need to set it unless required.

Other optional paremeters that can be set include startcol, startrow, engine, freeze_panes.

#saving the dataframe read to excel
import pandas as pd
data=excel_file=pd.read_excel("Admission_Predict.xlsx", sheet_name="Admission_Predict", header=0)
data.to_excel("testing.xlsx", index=False, sheet_name="data", float_format="%.2f")

Reading HDF5 files

HDF5 (Hierarchical Data Format version 5) is a file format and set of tools designed to store and manage large amounts of data. It is widely used in scientific computing and data-intensive applications because it supports the creation, access, and sharing of complex data.

Key features of HDF5 include:

Hierarchical Structure: Data is organized in a file in a hierarchical manner, similar to a filesystem. This allows the grouping of related datasets and metadata in a structured way.

Scalability: HDF5 can efficiently handle very large datasets, both in terms of size and complexity. It is optimized for high performance in both reading and writing operations.

Portability: HDF5 files are designed to be platform-independent, meaning they can be created and read on different types of systems without compatibility issues.

Flexibility: It supports various data types and complex data relationships, including multidimensional arrays, tables, and complex objects. This makes it suitable for a wide range of applications.

Metadata Storage: HDF5 allows the storage of rich metadata along with the data, which is crucial for documenting and understanding the data.

Compression: The format supports compression and other storage optimizations to save space and improve performance.

Parallel I/O: HDF5 provides support for parallel I/O, allowing multiple processes to read/write data simultaneously, which is important for high-performance computing environments.

HDF5 is commonly used in fields such as astronomy, physics, engineering, bioinformatics, and many other scientific domains where managing large and complex datasets is essential. It is supported by various programming languages, including Python (with the h5py library), C, C++, Fortran, and others.

Pandas provides the read_hdf() for reading HDF5 files.

Interacting with web APIs

In some cases, organizations provides access to data using an API. An API in full is Application Programming Interfaces. It provides for an interface for applications to be able to communicate with one another.

There are different types of APIs for different purposes

Web APIs: Web APIs are APIs that provide transfer of machine readable data between web based applications. There are different formats used in web APIs. This section reviews a few of them

REST (Representational State Transfer): Uses HTTP requests to GET, POST, PUT, DELETE data. It's stateless and often returns data in JSON or XML format.

SOAP (Simple Object Access Protocol): A protocol for exchanging structured information in the implementation of web services. It uses XML and can operate over various protocols such as HTTP, SMTP.

GraphQL: Developed by Facebook, it allows clients to request exactly the data they need. It provides a more flexible and efficient alternative to REST.

Example

In this example we are accessing the weather data provided by the open weather map using a REST API. You have to create an acount in the openweather app in order to get the API key. Once you have the API key you can use it for extracting the weather data from the site. Then we can save the extracted data into the pandas dataframe.

import requests

# Define your API key
api_key = 'f4aaacf29976bba0a688d364c5e511c3'

# Define the API endpoint
base_url = 'https://api.openweathermap.org/data/2.5/weather'

# Define the parameters
lat = 1.2921   # Latitude for Nairobi
lon = -36.8219   # Longitude for Nairobi
exclude = 'minutely,hourly'  # Parts to exclude from the response (optional)

# Set up the parameters dictionary
params = {
    'lat': lat,
    'lon': lon,
    'exclude': exclude,
    'units': 'metric',  # To get temperature in Celsius
    'appid': api_key,
}

# Make the GET request to the API
response = requests.get(base_url, params=params)

# Print the full URL for debugging
print(f'URL: {response.url}')

# Check if the request was successful
if response.status_code == 200:
    # Parse the JSON response
    weather_data = response.json()
    # Extract relevant information (e.g., current temperature)
    current_temp = weather_data['main']['temp']

    print(f'The current temperature in Nairobi is {current_temp}°C with')
else:
    print(f'Failed to retrieve data: {response.status_code}')
    # Print the response text for debugging
    print(response.text)

Handling binary data files

Pandas provides efficient methods for handling binary file formats, which are useful for storing and retrieving data in a compact, fast, and type-preserving way. The primary binary formats supported by Pandas are Pickle, HDF5, and Parquet. In this section we are going to look at how to use this binary formats.

Pickle is a Python-specific binary format that allows for the serialization and deserialization of Python objects.

To save a dataframe to a picke file, we will use the to_pickle() function as follows

import pandas as pd

# Create a sample DataFrame
df = pd.DataFrame({
    'Name': ['Alice', 'Bob', 'Charlie'],
    'Age': [25, 30, 35]
})

# Save the DataFrame to a Pickle file
df.to_pickle('data.pkl')

If the path is not specified the pickle file will be saved in the location where your notebook is. To read data from a pickle file and save it into a dataframe, we will use the read_pickle() function. We are going to use the pickle file saved in the previous cell to demontsrate reading the data from a pickle file.

# Read the DataFrame from the Pickle file
df = pd.read_pickle('data.pkl')

# Display the DataFrame
print(df)
